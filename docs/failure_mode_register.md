# Failure Mode Register

| Failure Mode | Risk | Mitigation |
|---|---|---|
| Formula demo only | Project reads as a simple score formula. | Center the action-set diff and reviewer walkthrough. |
| Tuned lambda | Lambda looks chosen to force a dramatic result. | Always report `0.0`, `0.5`, `1.0`, and zero-uncertainty control. |
| Synthetic fixture brittleness | Scores look authored to prove the point. | Include clean and messier fixtures with survivor and non-entry cases. |
| Operational overclaim | Project sounds like real WTCHP decision advice. | Use synthetic, method-centered framing only. |
| Assertion leakage | Computed IDs are copied from expected IDs. | Include a self-comparison guard and fail-closed expected-value handling. |
| Scope sprawl | Build expands into dashboard, model, or platform. | Keep v1 deterministic and artifact-centered. |
